The model is configuration on the agent — not part of code or a runtime
image. A change takes effect on the next model call — the next turn, when
switching providers; nothing restarts.

<Note>**Preview** — the hot-swap and metering behavior on this page runs through
the model gateway, which is **not enabled in v0** (see the
[overview](/background-agents/overview)). Today the `claude` runtime calls the
provider directly with a configured key; model addressing and per-agent
configuration are real, but live rewrite, cross-provider flips, and gateway
metering are the designed behavior.</Note>

## Addressing

Provider-prefixed, provider-native IDs:

```
anthropic/claude-opus-4-8
openai/gpt-5.x
openrouter/qwen-3-coder
```

OpenComputer adds no model names of its own. Bare IDs are accepted when
unambiguous.

```bash
oc model list     # every model available through your connected providers
```

## Switching

```bash
oc agent update fixer --model openai/gpt-5.x    # new agent version
oc session set <id> --model openai/gpt-5.x      # one session, in flight
```

Agent-level changes are versions; session-level overrides appear as `status`
events on the timeline; usage records the model that served each call.
Switching to a different provider requires a [runtime](/background-agents/runtimes)
that drives it (the planned `opencode` drives all — Preview).

## Provider keys

```bash
oc connect anthropic --api-key sk-ant-...
oc connect openai --api-key sk-...
oc connect openrouter --api-key sk-or-...
```

Bring your own keys per provider, or use platform billing where available.
One OpenRouter key opens its full multi-lab catalog.

As designed, keys never enter agent machines: model traffic flows through
OpenComputer's model gateway, which holds the keys, enforces the configured
model, and meters tokens per session. The gateway is a Preview capability —
see the [overview](/background-agents/overview).

## Restricting

Organizations can restrict allowed providers
([policies](/background-agents/agents#policies)); agent data goes only to
providers you allow.
