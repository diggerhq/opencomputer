After this page, labeling a GitHub issue `agent` produces a draft PR, and
commenting on that PR wakes the same agent to follow up.

## 1. Connect GitHub

```bash
oc connect github
```

Installs the OpenComputer GitHub App on your org. No app registration,
webhook setup, or secrets.

## 2. Create a workspace

```bash
oc workspace create web \
  --repo github.com/acme/web \
  --setup "pnpm i && pnpm build"
```

The first build clones the repo, runs setup, and snapshots the result. After
that it rebuilds on every push to the default branch, so sessions start from
a ready machine matching the latest code. `oc workspace builds web` lists
the builds.

## 3. Create the agent

```bash
oc agent create fixer \
  --workspace web \
  --model anthropic/claude-opus-4-8 \
  --prompt "You fix bugs labeled 'agent' in acme/web. Keep diffs minimal. Run the affected tests before pushing." \
  --trigger github:issue.labeled:agent
```

## Use it

Label an issue `agent`. A session starts within seconds:

```bash
oc session list --agent fixer
oc session attach <session-id>   # live events; Ctrl-C detaches, the agent keeps working
```

A draft PR opens, attributed to whoever labeled the issue. The session goes
dormant; `oc session get <id>` shows `usage.active_seconds` — dormant time
uses no compute.

Comment on the PR: *"also update the changelog."* The session wakes with its
full context, pushes a commit, and sleeps again. The thread persists; a
comment weeks later wakes the same context.

## Change the model

```bash
oc agent update fixer --model openai/gpt-5.x
```

Effective when the agent next wakes — switching providers means the runtime
reconnects; a change within the same provider applies on the very next model
call. Same workspace, prompt, and triggers. Each change is a new agent
version — `oc agent rollback fixer v1` reverts.
