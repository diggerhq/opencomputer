By the end of this page, labeling a GitHub issue `agent` will produce a
draft PR, and commenting on that PR will wake the same agent to follow up.

## 1. Connect GitHub

```bash
oc connect github
```

Opens the GitHub App installation flow for your org. That's the whole
integration — no app registration, no webhook configuration, no secrets to
copy.

## 2. Create a workspace

A workspace is the prepared machine your agent works in:

```bash
oc workspace create web \
  --repo github.com/acme/web \
  --setup "pnpm i && pnpm build" \
  --refresh 30m --warm 1
```

The first build clones the repo, runs your setup, and snapshots the result
(a few minutes). From then on it rebuilds itself every 30 minutes, so
sessions always start from a machine that's at most half an hour stale —
with dependencies installed and the build warm.

```bash
oc workspace builds web    # watch the build lineage accumulate
```

## 3. Create the agent

An agent can start as just a prompt:

```bash
oc agent create fixer \
  --workspace web \
  --model anthropic/claude-opus-4-8 \
  --prompt "You fix bugs labeled 'agent' in acme/web. Keep diffs minimal and focused. Run the affected tests before pushing." \
  --trigger github:issue.labeled:agent
```

Done. There is no step 4 — no server, no queue, no code.

## Use it

Label any issue in `acme/web` with `agent`. Within seconds a session starts;
watch it live if you like:

```bash
oc session list --agent fixer
oc session attach <session-id>     # live event stream; Ctrl-C detaches, agent keeps working
```

A draft PR appears, opened on behalf of whoever labeled the issue. The
session then goes **dormant** — check `oc session get <id>` and note
`usage.active_seconds`: you were billed for minutes of work, and dormancy
costs ~nothing.

Now comment on the PR: *"also update the changelog."* The dormant session
wakes with full context — the same conversation, the same machine state —
pushes a commit, and sleeps again. That thread can stay alive as long as the
PR does.

## Change the model

```bash
oc agent update fixer --model openai/gpt-5.x
```

Same agent, same workspace, same triggers — the next model call runs on the
other provider. Switch back any time; every change is a new agent version
you can roll back.
