title: "Durable Agent Sessions"
description: "Define an agent, then run resumable, steerable sessions"
---

Give an agent a prompt and a task; OpenComputer runs it on a **fully managed runtime** and records **every step as a durable, streamable, resumable session**. Watch it work live, drop your connection and reconnect right where you left off, let it idle and the sandbox hibernates (≈ free), then steer it with a message or get notified by webhook.

**You never babysit the runtime.** A session is *durable*: we keep your agent running, and recover it for you.

- **Crashes or hangs?** We **restart it automatically** — it continues from where it was, mid-task, not from scratch.
- **Idle for hours or days?** It hibernates (≈ free) and **survives even if the underlying sandbox is reclaimed** — its state is checkpointed off-box, so it wakes back up intact.
- **Disconnect?** Reconnect from any cursor — nothing missed.

That managed, self-healing runtime — not just a chat API over a model — is the point.

<Steps>
  <Step title="Define an agent (with your key)">
    A reusable `{ name, model, prompt, runtime }` — `POST /v3/agents`. Pass your Anthropic `key` inline (sessions run on your own key); it's stored as a [credential](/agent-sessions/authentication#model-credentials). ([Agents](/agent-sessions/agents))
  </Step>
  <Step title="Start a session">
    `POST /v3/sessions { agent, input }` and the agent starts working immediately; you get back the session and a browser-safe `client_token`.
  </Step>
  <Step title="Watch, steer, get notified">
    Stream `GET …/events?stream=sse` — reconnect from any cursor to [resume](/agent-sessions/events). `POST …/messages` to [steer](/agent-sessions/messaging), and register a [webhook](/agent-sessions/webhooks) to have output pushed to you.
  </Step>
</Steps>

Every session hands back a scoped [client token](/agent-sessions/authentication#client-tokens) (`read` + `steer`), so a front-end can stream and steer **without** your API key.

## What the platform handles for you

The hard parts, handled:

<CardGroup cols={2}>
  <Card title="Self-healing, managed runtime" icon="heart-pulse">
    - Crash or hang → **auto-restart**, continues mid-task
    - Idle → hibernate (≈ free); **survives sandbox loss**
    - You never restart anything — [details](/agent-sessions/durability)
  </Card>
  <Card title="Brain / hands isolation" icon="shield-halved">
    - The **brain** (agent loop) runs apart from the **hands** sandbox (its files + commands)
    - Untrusted work stays contained
    - Your model key never enters either sandbox — it lives in the [secret store](/sandboxes/secrets)
  </Card>
  <Card title="Durable log & resume" icon="database">
    - Every step in one ordered, durable log
    - Reconnect from any cursor — nothing missed
  </Card>
  <Card title="Reliable delivery" icon="paper-plane">
    - At-least-once, retried, dead-lettered
    - Signed ([Standard Webhooks](https://www.standardwebhooks.com)); dedupe on `webhook-id`
  </Card>
</CardGroup>

## When to use it

Reach for sessions when you want a long-running or interactive agent and don't want to build the plumbing above yourself. For a one-shot command in a raw box, use a [sandbox](/sandboxes/overview) directly.
