---
title: "Durable Agent Sessions"
description: "Define an agent, then run resumable, steerable sessions"
---

Give an agent a prompt and a task; OpenComputer runs it on a managed runtime in a sandbox and records **every step as a durable, streamable, resumable session**. Watch it work live, drop your connection and reconnect right where you left off, let it idle and the sandbox hibernates (≈ free), then steer it with a message or get notified by webhook.

<Steps>
  <Step title="Define an agent">
    A reusable `{ name, model, prompt, runtime }` — `POST /v3/agents`. ([Agents](/agent-sessions/agents))
  </Step>
  <Step title="Start a session">
    `POST /v3/sessions { agent, input }` and the agent starts working immediately; you get back the session and a browser-safe `client_token`. *(Or inline `{ prompt, model, runtime, input }` for a one-off, without a reusable agent.)*
  </Step>
  <Step title="Watch, steer, get notified">
    Stream `GET …/events?stream=sse` — reconnect from any cursor to [resume](/agent-sessions/events). `POST …/messages` to [steer](/agent-sessions/messaging), and register a [webhook](/agent-sessions/webhooks) to have output pushed to you.
  </Step>
</Steps>

Every session hands back a scoped [client token](/agent-sessions/authentication#client-tokens) (`read` + `steer`), so a front-end can stream and steer **without** your API key.

## What the platform handles for you

The non-trivial parts you'd otherwise build — and get subtly wrong — handled:

<CardGroup cols={2}>
  <Card title="Reliable delivery" icon="paper-plane">
    Outbound messages are delivered at-least-once, retried with backoff, and dead-lettered — nothing silently dropped. Dedupe on `X-OC-Event-ID`.
  </Card>
  <Card title="Managed sandbox lifecycle" icon="box">
    Provision, hibernate on idle (storage-only ≈ free), wake in milliseconds, replace an unresponsive sandbox, tear down — none of it your problem.
  </Card>
  <Card title="Isolated execution" icon="shield-halved">
    The agent's loop runs separately from the sandbox where its file changes and commands execute — the untrusted work is contained, apart from the loop and your model credentials.
  </Card>
  <Card title="Durable log & resume" icon="database">
    Every step is recorded in a durable event log. Reconnect from any point and resume exactly where you left off — nothing lost or replayed.
  </Card>
</CardGroup>

## When to use it

Reach for sessions when you want a long-running or interactive agent and don't want to build the plumbing above yourself. For a one-shot command in a raw box, use a [sandbox](/sandboxes/overview) directly.
