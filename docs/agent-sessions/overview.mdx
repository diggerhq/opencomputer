Give an agent a prompt and a task; OpenComputer runs it on a **fully managed, self-healing runtime** and records every step as a **durable, streamable, resumable session** — watch it live, steer it, get notified by webhook. A managed agent runtime, not just a chat API over a model.

## What the platform handles for you

<CardGroup cols={2}>
  <Card title="Self-healing, managed runtime" icon="heart-pulse">
    - Crash → **auto-restart**, picks up where it was
    - Idle → hibernate (≈ free); **session survives sandbox loss**
    - You never restart anything — [details](/agent-sessions/durability)
  </Card>
  <Card title="Brain / hands isolation" icon="shield-halved">
    - The **brain** (agent loop) runs apart from the **hands** sandbox (its files + commands)
    - Untrusted work stays contained
    - Your model key never enters either sandbox — it lives in the [secret store](/sandboxes/secrets)
  </Card>
  <Card title="Durable log & resume" icon="database">
    - Every step in one ordered, durable log
    - Reconnect from any cursor — nothing missed
  </Card>
  <Card title="Reliable delivery" icon="paper-plane">
    - At-least-once, retried, dead-lettered
    - Signed ([Standard Webhooks](https://www.standardwebhooks.com)); dedupe on `webhook-id`
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Define an agent (with your key)">
    A reusable `{ name, model, prompt, runtime }` — `POST /v3/agents`. Pass your Anthropic `key` inline; it's stored as a [credential](/agent-sessions/authentication#model-credentials). ([Agents](/agent-sessions/agents))
  </Step>
  <Step title="Start a session">
    `POST /v3/sessions { agent, input }` — the agent starts immediately; you get the session and a browser-safe `client_token`.
  </Step>
  <Step title="Watch, steer, get notified">
    Stream `GET …/events?stream=sse` ([resume](/agent-sessions/events) from any cursor), `POST …/messages` to [steer](/agent-sessions/messaging), and register a [webhook](/agent-sessions/webhooks) for push.
  </Step>
</Steps>

## How it works

<Frame>
  <img src="/images/durable-agent-sessions-architecture.svg" alt="Durable Agent Sessions architecture: your app starts/steers and streams a durable event-log session; a runtime (the brain) reads and appends to it and acts in a sandbox (the hands); the model runs on your key from the secret store, which never enters a sandbox; user-level events are delivered out via your webhook." />
</Frame>

The **event log** is the durable spine: you start and steer against it, stream from it, and it drives the runtime. The **brain** (agent loop) is isolated from the **hands** sandbox (where commands and file edits run), your key lives in the secret store — never in a sandbox — and user-level events are delivered out to your webhook.

## When to use it

Reach for sessions when you want a long-running or interactive agent and don't want to build the plumbing above yourself. For a one-shot command in a raw box, use a [sandbox](/sandboxes/overview) directly.
