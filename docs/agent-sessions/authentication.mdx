Three credentials, each with a job.

## Org API key

Your account key. Use it server-side for all **management** calls ‚Äî creating agents, starting sessions, registering webhooks, managing credentials.

```bash
curl https://api.opencomputer.dev/v3/sessions \
  -H "Authorization: Bearer $OPENCOMPUTER_API_KEY" -d '{ ‚Ä¶ }'
```

<Warning>Never ship the org API key to a browser or mobile client ‚Äî it can do anything on your account. Use a client token instead.</Warning>

## Client tokens

A **session-scoped, short-lived** token with `read` + `steer` scope ‚Äî safe to hand to a front-end. It can stream and steer **one** session and nothing else.

You get one back from `POST /v3/sessions`, and can mint more:

```bash
curl https://api.opencomputer.dev/v3/sessions/$ID/client-tokens \
  -H "Authorization: Bearer $OPENCOMPUTER_API_KEY" \
  -d '{ "scopes": ["read", "steer"], "ttl": 3600 }'
```

The pattern: your backend creates the session with the org key, then hands the `client_token` to the browser, which uses it directly for the SSE stream and steer calls.

<Warning>A client token can **spend**: each `steer` starts a turn, which incurs model cost on the session it's scoped to. Mint short-lived tokens, and rely on the session's [bounds](/agent-sessions/sessions#bounds) (`max_usd`/`max_turn_seconds`/`max_turns`) to cap exposure.</Warning>

```
browser ‚îÄ‚îÄ(create task)‚îÄ‚îÄ‚ñ∂ your backend ‚îÄ‚îÄ(org key)‚îÄ‚îÄ‚ñ∂ POST /v3/sessions
browser ‚óÄ‚îÄ‚îÄ(client_token)‚îÄ‚îÄ your backend
browser ‚îÄ‚îÄ(client_token)‚îÄ‚îÄ‚ñ∂ GET ‚Ä¶/events (SSE) ¬∑ POST ‚Ä¶/messages
```

For the SSE stream the browser passes the client token as `?token=` (native `EventSource` can't set an `Authorization` header); other calls use the `Authorization` header normally.

## Model credentials (BYO key)

Model usage is **billed to your OpenComputer account** by default ‚Äî zero setup, metered, capped per session. To run on **your own** provider key instead, add a **credential**. It's stored in our secret store (encrypted, write-only, never returned) ‚Äî you never manage a raw secret store yourself.

```bash
curl https://api.opencomputer.dev/v3/credentials \
  -H "Authorization: Bearer $OPENCOMPUTER_API_KEY" \
  -d '{ "provider": "anthropic", "key": "sk-ant-‚Ä¶", "name": "prod" }'
# ‚Üí { "id": "cred_‚Ä¶", "provider": "anthropic", "last4": "‚Ä¶", ‚Ä¶ }

# make it the default for this provider
curl -X PUT https://api.opencomputer.dev/v3/credentials/default \
  -H "Authorization: Bearer $OPENCOMPUTER_API_KEY" \
  -d '{ "provider": "anthropic", "credential": "cred_‚Ä¶" }'
```

**Resolution** per session: the agent's credential ‚Üí your org default for the model's provider ‚Üí platform billing.

<Note>Additional model providers and model hot-swap are coming soon ‚Äî they use this same credential shape, so your setup won't change.</Note>
