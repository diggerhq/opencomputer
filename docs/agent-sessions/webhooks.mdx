Register a **destination** and a session's outbound messages are delivered to it over the shared [envelope](/agent-sessions/messaging#the-envelope) — at-least-once, signed, and retried. Filter what you receive so you're not spammed with internal chatter.

## Destinations

```bash
curl https://api.opencomputer.dev/v3/sessions/$ID/destinations \
  -H "Authorization: Bearer $OPENCOMPUTER_API_KEY" \
  -d '{ "url": "https://your.app/oc-webhook",
        "secret": "whsec_…",
        "level": "user",          // user (default) | progress
        "kinds": ["message"],     // optional allow-list; default all
        "include_raw": false }'   // strip raw from the delivered envelope
```

| Method · Path | Purpose |
| --- | --- |
| `POST /sessions/:id/destinations` | Register a destination. |
| `GET /sessions/:id/destinations` · `GET …/:did` | List / fetch. |
| `PATCH /sessions/:id/destinations/:did` | Pause or resume (`enabled`), retune `level`/`kinds`, or rotate `secret`. |
| `DELETE /sessions/:id/destinations/:did` | Remove. |

## Deliveries

Every send and its outcome — the data behind a deliveries dashboard.

| Method · Path | Purpose |
| --- | --- |
| `GET /sessions/:id/deliveries?destination=&status=` | List deliveries — status, attempts, last response/error. |
| `GET /sessions/:id/deliveries/:id` | One delivery, in detail. |
| `POST /sessions/:id/deliveries/:id/redeliver` | Re-send — **any** delivery, not just dead-lettered. |

A delivery moves `pending → delivering → delivered`, or `failed` (retrying) → `dead_letter` after the max attempts.

```
Delivery = { id, destination_id, event_id, status, attempts, last_attempt_at, response_code?, error? }
```

## Delivery contract

- **HTTPS only.** Private, loopback, link-local, and cloud-metadata addresses are refused.
- **At-least-once, idempotent per (event × destination)**, with exponential backoff; after the max attempts a delivery is **dead-lettered** (inspect and redeliver it above).
- **Signed.** Each request carries `X-OC-Signature` (HMAC-SHA256 over `timestamp + "." + body`, keyed by your `secret`) and `X-OC-Timestamp`. The `secret` is write-only — set once, never returned.
- A delivery succeeds on HTTP `2xx`.

### Verify a webhook

```js
import { createHmac, timingSafeEqual } from "node:crypto";

function verify(req, secret, body) {
  const ts = req.headers["x-oc-timestamp"];
  if (Math.abs(Date.now() / 1000 - Number(ts)) > 300) return false; // reject stale → replay guard
  const expected = createHmac("sha256", secret).update(`${ts}.${body}`).digest("hex");
  const got = req.headers["x-oc-signature"];
  return got.length === expected.length &&
         timingSafeEqual(Buffer.from(got), Buffer.from(expected));
}
```
