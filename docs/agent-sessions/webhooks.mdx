Register a **destination** and a session's outbound messages are delivered to it over the shared [envelope](/agent-sessions/messaging#the-envelope) — at-least-once, signed, and retried. Filter what you receive so you're not spammed with internal chatter.

## Destinations

```bash
curl https://api.opencomputer.dev/v3/sessions/$ID/destinations \
  -H "Authorization: Bearer $OPENCOMPUTER_API_KEY" \
  -d '{ "url": "https://your.app/oc-webhook",
        "secret": "whsec_…",
        "level": "user",
        "include_raw": false }'
```

`level` is the visibility threshold (`user` default, or `progress`); `types` is an optional event-type allow-list (default: all — which **includes the terminal `turn.completed`**); `include_raw: false` strips `raw` from the delivered envelope. To get only completions, set `types: ["turn.completed"]`.

| Method · Path | Purpose |
| --- | --- |
| `POST /sessions/:id/destinations` | Register a destination. |
| `GET /sessions/:id/destinations` · `GET …/:did` | List / fetch. |
| `PATCH /sessions/:id/destinations/:did` | Pause or resume (`enabled`), retune `level`/`types`, or rotate `secret`. |
| `DELETE /sessions/:id/destinations/:did` | Remove. |

A destination receives events appended **after** it's created. To capture a whole session, pass `webhook` in `POST /v3/sessions` so it's registered before the agent produces output. (Replaying earlier events with `backfill_from_seq` is coming soon.)

## Deliveries

Every send and its outcome — the data behind a deliveries dashboard.

| Method · Path | Purpose |
| --- | --- |
| `GET /sessions/:id/deliveries?destination=&status=` | List deliveries — status, attempts, last response/error. |
| `GET /sessions/:id/deliveries/:id` | One delivery, in detail. |
| `POST /sessions/:id/deliveries/:id/redeliver` | Re-send — **any** delivery, not just dead-lettered. |

A delivery moves `pending → delivering → delivered`, or `failed` (retrying) → `dead_letter` after the max attempts.

```
Delivery = { id, destination_id, event_id, status, attempts, last_attempt_at, response_code?, error? }
```

## Delivery contract

- **HTTPS only.** Private, loopback, link-local, and cloud-metadata addresses are refused.
- **At-least-once.** Retried with exponential backoff; after the max attempts a delivery is **dead-lettered** (inspect and redeliver it above). A duplicate can still arrive — a send that succeeds as the worker dies, or a replayed turn — so **dedupe on `X-OC-Event-ID`**.
- **Correlation headers.** Every request carries `X-OC-Event-ID`, `X-OC-Delivery-ID`, and `X-OC-Session-ID` (for dedupe and correlation), plus the signature headers below.
- **Signed.** `X-OC-Signature` = HMAC-SHA256 over `X-OC-Timestamp + "." + rawBody`, keyed by your `secret`. Verify against the **raw request body, before parsing it**. `X-OC-Timestamp` guards replay; the `secret` is write-only — set once, never returned.
- A delivery succeeds on HTTP `2xx`.

### Verify a webhook

```js
import { createHmac, timingSafeEqual } from "node:crypto";

// body = the RAW request body string, read BEFORE JSON.parse
function verify(req, secret, body) {
  const ts  = req.headers["x-oc-timestamp"];
  const got = req.headers["x-oc-signature"];
  if (!ts || !got) return false;                                         // missing headers → reject
  if (!/^\d+$/.test(ts) || Math.abs(Date.now() / 1000 - Number(ts)) > 300) return false; // bad/stale ts → replay guard
  const expected = createHmac("sha256", secret).update(`${ts}.${body}`).digest("hex");
  let a, b;
  try { a = Buffer.from(got, "hex"); b = Buffer.from(expected, "hex"); }  // decode hex explicitly
  catch { return false; }                                                // malformed signature
  return a.length === b.length && timingSafeEqual(a, b);                  // constant-time compare
}
```

<Note>Today, delivery is over **generic webhooks**. Slack, GitHub comments, Jira, and email are coming soon — all on this same envelope and delivery contract.</Note>
