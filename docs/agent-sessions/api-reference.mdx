Base URL `https://api.opencomputer.dev/v3`. Management calls authenticate with your **org API key**; stream and steer also accept a **[client token](/agent-sessions/authentication#client-tokens)**.

## Sessions

| Method · Path | Purpose |
| --- | --- |
| `POST /sessions` | Start a session. Body: `agent?`, `prompt?`, `model?`, `runtime?`, `input`, `key?`, `webhook?`, `destinations?[]`, `max_usd?`, `max_turn_seconds?`, `max_turns?`. Returns `{ session, client_token }`. |
| `GET /sessions/:id` | Fetch a session. |
| `GET /sessions?status=&agent=&key=&after=&before=&limit=` | List sessions (filtered, paginated). |
| `POST /sessions/:id/archive` | Cancel any active turn, then close (read-only). |
| `POST /sessions/:id/cancel` | Cooperatively stop the active turn (no-op if not running). |
| `GET /sessions/:id/result` | The result helper — `{ last_turn, result? }` (the resolved final event). |
| `GET /sessions/:id/turns` · `GET …/turns/:turnId` | Per-turn history: timing, usage, error. |
| `POST /sessions/:id/client-tokens` | Mint a client token. Body: `scopes?`, `ttl?`. |

Provide `agent` (a saved agent id) **or** inline `prompt` + `model` + `runtime`. `key` gives get-or-create (one session per key); a request-level `Idempotency-Key` header makes a keyless create retry-safe. `input` is the task — a string or a full [envelope](/agent-sessions/messaging#the-envelope) (structured context rides `input.refs`: `repo`/`branch`/`commit`/`issue`/…). The bounds `max_usd` / `max_turn_seconds` / `max_turns` cap spend, per-turn wall-clock, and auto-runs; hitting one ends the turn with a matching `last_turn.yield_reason`.

```
Session = { id, status, head, cursor,
            agent_snapshot: { prompt_hash, model, runtime },
            last_turn: { state, yield_reason?, turn_id, started_at, completed_at?, result_event_id?, error?, budget_exceeded? },
            usage: { active_seconds, usd, cap_usd?, tokens? },
            limits: { max_usd?, max_turn_seconds?, max_turns? }, key?, created_at }
```

`yield_reason` (`completed | needs_input | error | budget_exceeded | deadline_exceeded | max_turns | canceled`) is the machine-readable "why the turn ended" — `needs_input` means the agent asked a question; answer it by [steering](#steer).

## Events

| Method · Path | Purpose |
| --- | --- |
| `GET /sessions/:id/events?after=<seq>&level=<lvl>&kind=&type=&turn_id=&limit=<n>` | Read events since a cursor (filter by `kind`, `body.type`, or `turn_id`). |
| `GET /sessions/:id/events?after=<seq>&level=<lvl>&stream=sse` | Live SSE stream from a cursor. |
| `GET /sessions/:id/events/:eventId` | Fetch one event (resolve `result_event_id`). |
| `GET /sessions/:id/events/:eventId/content` | Fetch a blob-backed body (large outputs/artifacts). |
| `GET /sessions/:id/messages?after=<seq>&limit=<n>` | User-message history (alias for `level=user` + `kind=message`). |

`after` resumes from a `seq`; `level` is a **visibility threshold** — `user` returns only user-facing events, `progress` adds work updates, `internal` adds everything (see [levels](/agent-sessions/events#levels)). Switch on [`body.type`](/agent-sessions/events#event-shape), don't parse prose.

```
Event = { id, seq, ts, session, actor, kind, level, body, refs, operation_key? }
```

## Steer

| Method · Path | Purpose |
| --- | --- |
| `POST /sessions/:id/messages` | Append an input message; wakes the session. Body: `{ text, idempotency_key? }` or `{ envelope }`. Returns `{ seq }`. |

## Destinations

| Method · Path | Purpose |
| --- | --- |
| `POST /sessions/:id/destinations` | Register a webhook. Body: `url`, `secret?`, `level?` (`user`), `kinds?[]`, `include_raw?` (`false`), `enabled?` (`true`). |
| `GET /sessions/:id/destinations` · `GET …/:did` | List / fetch. |
| `PATCH /sessions/:id/destinations/:did` | Pause/resume (`enabled`), retune `level`/`kinds`, rotate `secret`. |
| `DELETE /sessions/:id/destinations/:did` | Remove. |

## Deliveries

| Method · Path | Purpose |
| --- | --- |
| `GET /sessions/:id/deliveries?destination=&status=` | List deliveries — status, attempts, last response/error. |
| `GET /sessions/:id/deliveries/:id` | One delivery, in detail. |
| `POST /sessions/:id/deliveries/:id/redeliver` | Re-send any delivery (not just dead-lettered). |

See [Webhooks](/agent-sessions/webhooks#delivery-contract) for the signing and delivery contract.

```
Destination = { id, url, level, kinds, include_raw, enabled, created_at }
Delivery    = { id, destination_id, event_id, status, attempts, last_attempt_at, response_code?, error? }
```

## Agents

| Method · Path | Purpose |
| --- | --- |
| `POST /agents` | Create a reusable agent. Body: `name`, `prompt`, `model`, `runtime?` (`claude`). Idempotent by `(owner, name)`. |
| `GET /agents/:id` · `GET /agents` | Fetch / list. |
| `PATCH /agents/:id` | Update `prompt?` / `model?`. Never affects existing sessions (snapshots are pinned). |

## Credentials

| Method · Path | Purpose |
| --- | --- |
| `POST /credentials` | Add a provider key. Body: `provider`, `key`, `name?`. Key is write-only. |
| `GET /credentials` · `DELETE /credentials/:id` | List / remove. |
| `PUT /credentials/default` | Set the org default for a provider. Body: `provider`, `credential`. |

```
Credential  = { id, provider, name?, last4, created_at }
ClientToken = { token, scopes, expires_at }
```

## Errors & rate limits

Errors use one envelope: `{ "error": { "type", "message", "code", "param?", "request_id" } }`. Cross-owner resources return `404` (not `403`). Over a rate or concurrency limit, calls return **`429`** with a **`Retry-After`** header — back off and retry.
