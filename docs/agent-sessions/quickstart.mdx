## Prerequisites

```bash
export OPENCOMPUTER_API_KEY=your-api-key
```

<Tip>Grab your API key from [app.opencomputer.dev](https://app.opencomputer.dev).</Tip>

## 1. Start a session

You don't need to create an agent first — one call starts a session. Pass a `model`, a `prompt` (the instructions), and `input` (the task).

```bash
curl https://api.opencomputer.dev/v3/sessions \
  -H "Authorization: Bearer $OPENCOMPUTER_API_KEY" \
  -d '{ "model": "anthropic/claude-opus-4-8", "runtime": "claude",
        "prompt": "You are a coding assistant.",
        "input": "Clone github.com/acme/widget, run its tests, and summarize what fails." }'
```

```json
{
  "session": { "id": "sess_abc", "status": "running", "head": 0, "cursor": 0 },
  "client_token": "ct_…"     // session-scoped; safe to hand to a browser (read + steer)
}
```

<Note>Defining the agent inline like this is sugar for one-off sessions. To reuse an agent across sessions, [create one](/agent-sessions/agents) and start sessions with `{ "agent": "agt_…", "input": … }`.</Note>

## 2. Stream it live

Open an `EventSource` with the `client_token` from step 1. It rides the query string — browsers can't set headers on `EventSource`, so the scoped client token goes in the URL (never your API key). `level=user` shows only what an end user should see (see [levels](/agent-sessions/events#levels)).

```js
const es = new EventSource(
  `https://api.opencomputer.dev/v3/sessions/${id}/events?stream=sse&level=user&token=${clientToken}`
);
es.onmessage = (e) => render(JSON.parse(e.data)); // each event: { seq, actor, kind, level, body, … }
```

**Resume is automatic.** Each event carries its `seq` as the SSE event `id`, so on a dropped connection `EventSource` reconnects and resends `Last-Event-ID` — you pick up exactly where you left off, no bookkeeping. (Pass `after=<seq>` on the URL to start from a specific point.)

## 3. Steer it

Send a follow-up at any time. The session wakes and continues with full prior context.

```bash
curl https://api.opencomputer.dev/v3/sessions/$ID/messages \
  -H "Authorization: Bearer $CLIENT_TOKEN" \
  -d '{ "text": "also check the CI config", "idempotency_key": "nudge-1" }'
```

## 4. Get notified (optional)

Register a webhook and the agent's user-facing messages are delivered to it — signed, retried, and de-duplicated.

```bash
curl https://api.opencomputer.dev/v3/sessions/$ID/destinations \
  -H "Authorization: Bearer $OPENCOMPUTER_API_KEY" \
  -d '{ "url": "https://your.app/oc-webhook", "secret": "whsec_…", "level": "user" }'
```

See [Webhooks](/agent-sessions/webhooks#verify-a-webhook) to verify the signature.
