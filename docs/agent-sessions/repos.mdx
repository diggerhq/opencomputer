A **Repo** connects a Git repository to your sessions. The agent reads and edits the
checked-out code in its sandbox; authenticated GitHub operations (clone, fetch, push, PR,
comment) run outside it, with a short-lived token minted just in time — the agent never
holds a GitHub credential.

<Note>**Preview** — APIs may change before general availability.</Note>

## Connect GitHub

Connect once; OpenComputer mints short-lived, repo-scoped tokens just in time for each
operation.

| Method | Setup | Status |
| --- | --- | --- |
| **OpenComputer GitHub App** | install the App, select your repos | Recommended (landing) |
| Bring your own App — delegated key | register your App's id + private key | Coming soon |
| Bring your own App — broker | host a mint endpoint OpenComputer calls | Coming soon |
| Inline short-lived token | pass a token per session (can't refresh) | Available now (workaround) |

### OpenComputer GitHub App

Install the **OpenComputer** GitHub App and select the repositories agents may work in
(change the selection or uninstall anytime). No keys to manage, no endpoint to host.

Bring-your-own App (so PRs and comments appear as *your* App) is coming soon — either
register your App's key, or host a mint endpoint OpenComputer calls. Same repo model and
session API.

## Register a repo

Once GitHub is connected, register a repo — optional: any installed repo can be used in a
session directly; a Repo adds policy and a stable handle.

<CodeGroup>

```ts TypeScript SDK
const repo = await oc.repos.create({ owner: "acme", repo: "web" });
```

```http REST API
POST /v3/repos
Authorization: Bearer $OPENCOMPUTER_API_KEY

{ "provider": "github", "owner": "acme", "repo": "web" }
```

</CodeGroup>

Get-or-create, owner-scoped, idempotent by `(provider, owner, repo)`. **No credential is
passed** — auth comes from your GitHub connection. Optional `defaults`:
`{ branchNamespace, allow }` (`allow` is a GitHub permission ceiling).

## Use it in a session

```ts
const session = await oc.sessions.create({
  agent: agent.id,
  input: "Review this pull request.",
  sources: [{ repo: repo.id, ref: "refs/pull/42/head", sha: "abc123…", name: "head" }],
});
// checked out at /workspace/sources/head before the first turn
```

- **`ref`** (required) — the fetch ref (a branch or `refs/pull/N/head`); a private SHA
  can't be fetched directly.
- **`sha`** (required) — the exact commit, pinned and verified after fetch.

You can also reference `"owner/repo"` directly instead of a registered id. A session can
list several sources (a PR's base + head); they're **immutable**, and the agent pulls
more refs mid-session via the `source.fetch_ref` tool.

## Inline token — the no-setup workaround

Haven't connected the App yet? Pass a real short-lived token inline with the
deliberately-named `risky_short_lived_token` auth:

```ts
sources: [{ url: "https://github.com/acme/web.git", ref: "refs/pull/42/head", sha: "abc123…",
  auth: { type: "risky_short_lived_token", token: "ghs_…", expiresAt: "…" } }]
```

It works today but **can't refresh** — once the token expires the session can't fetch,
push, or recover (so it's only safe for sessions that finish within the token's life).
OpenComputer rejects a far-future/missing expiry, holds it **encrypted only until the
first checkout, then purges it**. This is a **temporary shortcut until the App-based
methods above land — prefer the GitHub App.**

## What the agent can do

Local Git works normally on the checkout (`status`/`diff`/`log`, edit, test, local
commits). Anything needing the network is a **typed tool** run in the isolated sandbox —
the agent never runs authenticated `git`/`gh` and never sees a token:

| Tool | Does | Status |
| --- | --- | --- |
| `source.fetch_ref` | fetch another ref into the checkout | Coming soon |
| `github.publish_branch` / `publish_pull_request` | push changes / open a PR | Coming soon |
| `github.comment` / `github.set_status` | comment, set a commit status | Coming soon |

## Guarantees

- **Connection-backed tokens are never stored** — OpenComputer mints short-lived,
  repo-scoped tokens just in time and destroys them. None reaches the agent's prompt,
  shell, files, env, the event log, or telemetry. (The one exception is the inline-token
  workaround, held encrypted only until the first checkout, then purged.)
- The GitHub App's private key lives only in OpenComputer's **control plane**, never in a
  sandbox.
- Agent pushes go to an `oc/<session>/…` branch namespace — **no protected-branch push,
  no force-push.**
- Submodules and Git LFS aren't supported yet. Lost-sandbox recovery and prebuilt
  checkouts for large repos ([Workspaces](/agent-sessions/workspaces)) are firming up.
