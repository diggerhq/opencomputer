A **session** is the durable unit of an [agent](/agent-sessions/agents) at work: an append-only [event log](/agent-sessions/events), the pinned agent snapshot, and a lifecycle status. Compute attaches to it, detaches when idle, and reattaches on the next message — the log is what persists.

## How a session works

1. You create a session from an agent (or an inline prompt + model). The session starts immediately.
2. The [runtime](/agent-sessions/runtimes) executes the agent in a sandbox, appending [events](/agent-sessions/events) to the log as it works.
3. With nothing left to do, the session goes **idle** and the sandbox **hibernates** (see [durability](/agent-sessions/durability)).
4. A [steer](/agent-sessions/messaging) message wakes it; it resumes with its prior context. Repeat.

<Note>If the runtime crashes mid-turn it auto re-wakes and continues — but in-flight work since the last logged event may repeat, so make side effects idempotent. Details: [durability](/agent-sessions/durability).</Note>

## Lifecycle

| Status | Meaning |
| --- | --- |
| `queued` | Scheduled; a turn hasn't started running yet. |
| `running` | A turn is executing. |
| `idle` | Quiescent and steerable; the sandbox is hibernated. |
| `failed` | The session errored out. |
| `archived` | Closed; read-only. |

A session also carries `last_turn = { state, yield_reason?, turn_id, started_at, completed_at?, result_event_id?, error?, budget_exceeded? }` and `usage` (`active_seconds`, `usd`). The **`yield_reason`** tells you *why* the last turn ended — `completed`, `needs_input` (the agent asked a question; answer by [steering](/agent-sessions/messaging)), `budget_exceeded` / `deadline_exceeded` / `max_turns` (a [bound](#bounds) tripped), or `canceled`.

## Get the result

Don't sniff prose for "is it done." Await the **`status` · `turn.completed`** event (on the [stream](/agent-sessions/events) or a [webhook](/agent-sessions/webhooks)), then fetch the answer:

```bash
curl https://api.opencomputer.dev/v3/sessions/$ID/result \
  -H "Authorization: Bearer $OPENCOMPUTER_API_KEY"
# → { "last_turn": { "state": "ok", "yield_reason": "completed", "result_event_id": "evt_…" },
#     "result": { "kind": "message", "body": { "type": "agent.message", "text": "…", "final": true } } }
```

## Bounds

Cap a session at create (or default it on the [agent](/agent-sessions/agents)): `max_usd`, `max_turn_seconds` (per-turn wall-clock), `max_turns`. Hitting one ends the turn with the matching `yield_reason`.

## Stop a session

`POST /sessions/:id/cancel` cooperatively stops the active turn and stops billing; `POST /sessions/:id/archive` cancels any active turn, then closes the session read-only.

<Note>Blocking on input mid-turn (formal `ask` / awaiting-input) is coming soon. Today the agent signals it needs input via `yield_reason: "needs_input"`, and you reply by [steering](/agent-sessions/messaging) — fire-and-continue.</Note>
