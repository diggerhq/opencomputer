An **agent** is a first-class, reusable definition of *what* runs — a `name`, a `prompt` (its instructions), a `model`, and a [`runtime`](/agent-sessions/runtimes). Every [session](/agent-sessions/sessions) runs on an agent, so create one first, then start as many sessions on it as you like.

```ts
const agent = await oc("/agents", {
  method: "POST",
  body: JSON.stringify({
    name: "reviewer",
    runtime: "claude",
    model: "anthropic/claude-opus-4-8",
    prompt: "Review the target repo. Run tests, summarize risks, and say the final result.",
    key: process.env.ANTHROPIC_API_KEY,
  }),
});
```

Pass your Anthropic `key` inline and it's stored as a [credential](/agent-sessions/authentication#model-credentials) and attached to the agent — sessions run the model on it. Already have a credential? Reference it with `credential: "cred_…"` instead, or omit both to use your org default. Optionally set default [`limits`](/agent-sessions/sessions#limits) (`tokens`/`turn_seconds`/`turns`) too; sessions inherit them.

<Tip>Use the server-side `oc()` helper from the [quickstart](/agent-sessions/quickstart#1-add-the-server-helper). Keep the org key and model key on your backend.</Tip>

## Pinned snapshot

When a session is created it **freezes** the agent's `prompt`, `model`, `runtime`, and `revision` (a number that bumps on every edit) into the session — and pins the resolved credential too. Editing the named agent afterwards never changes a running or resumed session — its behavior is pinned for its whole life. Update an agent freely: existing sessions keep what they started with, new sessions pick up the change. (Rotating a credential's **key value** is the exception — that flows to running sessions, so a compromised key can be replaced; see [authentication](/agent-sessions/authentication#model-credentials).)

## Model

`model` is a `provider/model` id (e.g. `anthropic/claude-opus-4-8`); the chosen [runtime](/agent-sessions/runtimes) determines which models are valid. The model runs on [your Anthropic key](/agent-sessions/authentication#model-credentials) (required). Today the only callable runtime is `claude`.

<Note>Additional model providers, and switching a session's model mid-session, are coming soon.</Note>
