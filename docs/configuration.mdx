## Environment Variables

Both SDKs read from these environment variables:

| Variable | Description | Default |
| --- | --- | --- |
| `OPENSANDBOX_API_KEY` | Your API key for authentication | — |
| `OPENSANDBOX_API_URL` | Base URL of the OpenSandbox API | `http://localhost:8080` |

```bash
export OPENSANDBOX_API_KEY="sk-your-api-key"
export OPENSANDBOX_API_URL="https://api.opensandbox.dev"
```

## Inline Configuration

You can also pass credentials directly when creating or connecting to a sandbox:

<CodeGroup>

```typescript TypeScript
import { Sandbox } from 'opensandbox';

const sandbox = await Sandbox.create({
  apiKey: 'sk-your-api-key',
  apiUrl: 'https://api.opensandbox.dev',
  template: 'base',
});
```

```python Python
from opensandbox import Sandbox

sandbox = await Sandbox.create(
    api_key='sk-your-api-key',
    api_url='https://api.opensandbox.dev',
    template='base',
)
```

</CodeGroup>

<Note>
  Inline configuration takes precedence over environment variables.
</Note>

## Authentication

All API requests are authenticated using your API key, sent as the `X-API-Key` header. Both SDKs handle this automatically when you provide the key via environment variable or inline config.

## Architecture: Server vs Combined Mode

OpenSandbox supports two deployment modes that affect how SDKs connect:

### Combined Mode (Default for Local Development)

In combined mode, a single process handles both the control plane and sandbox execution. The SDK talks to one URL for everything.

```
SDK → API Server (creates sandbox + runs commands)
```

### Server Mode (Production)

In server mode, the control plane and workers are separate. When you create a sandbox, the API returns a `connectURL` and `token`. The SDK then communicates directly with the worker for all data operations (commands, files, PTY).

```
SDK → Control Plane (creates sandbox, gets connectURL + token)
SDK → Worker (commands, files, PTY via connectURL)
```

This is handled transparently by the SDK — you don't need to change your code between modes.

## Sandbox Options

Full list of options available when creating a sandbox:

<CodeGroup>

```typescript TypeScript
interface SandboxOpts {
  template?: string;                    // Container template (default: 'base')
  timeout?: number;                     // Sandbox TTL in seconds (default: 300)
  apiKey?: string;                      // API key override
  apiUrl?: string;                      // API URL override
  envs?: Record<string, string>;        // Environment variables for the sandbox
  metadata?: Record<string, string>;    // Arbitrary key-value metadata
}
```

```python Python
# Sandbox.create() parameters
template: str = 'base'                  # Container template
timeout: int = 300                      # Sandbox TTL in seconds
api_key: str | None = None              # API key override
api_url: str | None = None              # API URL override
envs: dict[str, str] | None = None      # Environment variables for the sandbox
metadata: dict[str, str] | None = None  # Arbitrary key-value metadata
```

</CodeGroup>
